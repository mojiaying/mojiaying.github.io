                    # 1.ref

                    需求：两个兄弟元素，一个div,一个是button

                    button.onclick = function () {
    document.getElementById('xxx').style.background = ''
}

                    在react 页面元素都是动态生成的，无法使用DOM API

                    在React中，大部分操作的都是React元素，不是DOM元素，通常情况下，我们不需要，也不建议，直接操作原生的DOM元素。

                    ref: 用于访问在render方法中创建的React元素,就是获取真实DOM进行操作

                    谨慎使用Ref,能够使用属性和状态进行控制，就不要使用Ref

                    React里边，不要强制去修改DOM，而是通过props、state 重新渲染组件

                    import React, { createRef } from 'react'

/* 
  创建一个Ref对象 
  createRef()创建ref对象，并通过ref 属性添加至 React 元素上
  通常我们在构造函数中，分配给实例属性，可以方便在组件中使用
  ref => 目前是不可以的，函数组件没有实例
*/
constructor() {
    super()
    this.input = createRef()
    this.header = createRef()
    this.userRefs = createRef()
  }
  componentDidMount() {
    // 获取焦点
    this.input.current.focus()
  }

  render() {
    return (
      <div>
        {/* 用在组件上，得到的就是组件实例 */}
        <Header ref={this.header} />

        {/* 用于的是标签元素,获取的是dom对象 */}
        <h2 ref={this.userRefs}>H2标签</h2>
        <input type="text" ref={this.input} />
        <button onClick={this.changText}>按钮</button>
      </div>
    )
  }
changText = () => {
    console.log(this.input.current, this.header.current)

    // ref是一个对象，它有一个current属性操作原生DOM，这个属性用于获取DOM元素和保存变化的值
    console.log(this.header.current.state.name)

    // 调用子组件的方法
    this.header.current.say()

    // 赋值操作
    this.userRefs.current.innerHTML = this.input.current.value
  }

                    ref 转发：forwardRef父组件传来的ref对象绑在子组件元素上

                    import React, { PureComponent, createRef, forwardRef } from 'react'

const FocusInput = forwardRef(function (props, ref) {
  return <input type="text" ref={ref}></input>
})

export default class App extends PureComponent {
  constructor() {
    super()
    this.input = createRef()
  }

  componentDidMount() {
    this.input.current.focus()
  }


  render() {
    return (
      <div>
        <FocusInput ref={this.input}/>
      </div>
    )
  }
}

                    # 2.受控组件和非受控组件：都是表单元素

                    <input />
<textarea />
<select />

                    受控组件，表单元素受React组件管理控制，表单的状态（数据）修改只能通过setState()来更新

                    特点是：数据可控，完全由react中的state来管理

                     render() {
    const { name } = this.state
    return (
      <div>
        <h1>{name}</h1>
        <input type="text" value={name} onChange={this.changVal} />
        <hr />
        <button
          onClick={() => {
            console.log(`state:${name}`)
          }}
        >
          提交
        </button>
      </div>
    )
  }

  changVal = e => {
    this.setState({
      name: e.target.value
    })
  }

                    非受控组件：

                    使用Ref从DOM节点获取表单数据，表单数据交由DOM节点来处理

                     render() {
    const { name } = this.state
    return (
      <div>
        <h1>{name}</h1>
        <input
          type="text"
          value={name}
          onChange={this.changVal}
          ref={this.input}
        />
        <hr />
        <button
          onClick={() => {
            console.log(`数据:${this.input.current.value}`)
          }}
        >
          提交
        </button>
      </div>
    )
  }

  changVal = () => {
    this.setState({
      name: this.input.current.value
    })
  }

                    特点是：ref获取DOM节点进行数据更新，这样就不可控了，不能统一进行管理

                    # 3.高阶组件

                    高阶组件就是一个函数：

                    1. 接收一个或者多个函数作为参数
                    2. 输出一个函数（返回一个新的组件）

                    高阶组件就是通过props把account传递给目标组件了，目标组件就直接从props里边拿数据就可以了

                    **高阶组件解决的是功能的复用，**不是代码的复用

                    高阶组件的主要功能是封装并分离组件的通用逻辑，让通用的逻辑在组件间更好的复用

                    minxins：

                    React中，推崇的是组合，不推崇继承

                    minxin相互依赖，相互耦合，不利于代码的维护

                    minxin命名冲突

                    minxin非常多的时候，维护起来就很头疼

                    HOC需要在原来的组件上进行包裹或者嵌套，如果大量的使用HOC，将会产生非常多的嵌套，调试就很困难，HOC可以劫持props
